



##### 简单说明

```javascript
//算法是什么？
计算机程序执行的规则

//数据结构是什么？
信息的存储结构，算法的依赖

//为什么要用算法？算法的作用
做出最优决策，用最少的资源获取最大的效益，解决快和慢，多和少，先和后...的问题。

//如何判断最优决策？
通过复杂度分析

//数据结构与算法的关系
信息输入(数据)  ->  经过算法计算  ->  获取最优结果

//总结
1.数据结构是为算法服务的，算法是为决策服务的，反过来说，算法是基于数据结构的，决策是基于算法的
2.特定算法要作用在特定的数据结构上，通过复杂度分析来决定算法效率的优劣
```



##### 复杂度分析

```javascript
//计算机算法的复杂度分析依赖于两个维度：
时间：执行效率（快与慢）
空间：内存占用（大与小，多与少）
一般来说：最优的结果就是：执行效率快，占用内存小，也就是用了最少的资源，得到最好的结果。

//大O复杂度表示法：
O：复杂度衡量标志，用来计算一个算法的时间维度有多快，需要花费多少资源才能达成目标。

//复杂度计算的简单规则：
1.只关注循环执行次数最多的一段代码
2.加法法则：总复杂度等于量级最大的那段代码的复杂度
3.乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积（乘数总和）

//常见时间复杂度：
O(1)：常量级，动作只有一次，如：JS数组的push方法，在数组最后添加一个值
O(n)：线性型增长，动作有n次，如：JS数组的forEach方法，数组有多长，动作就有多少次
O(n^2)：n的2次方，循环嵌套，嵌套次数越多，数字越高，如：JS的两次for循环
```
